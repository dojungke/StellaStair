# Stella Stair

Unity 6 기반의 2D 횡스크롤 SRPG 프로젝트입니다. 현재 기반 시스템은 유닛 배치, 셀 점유, 이동 가능 범위 계산, A* 경로 이동, 공격과 적 AI 턴을 포함합니다.

## 실행

1. Unity Hub에서 이 폴더를 엽니다 (`6000.4.5f1`).
2. 상단 메뉴에서 `Stella Stair > Create Tactical Battle Scene`을 실행합니다.
3. 생성된 `Assets/StellaStair/Sample/TacticalBattle.unity` 씬에서 Play를 누릅니다.
4. 유닛을 클릭하고 파란 배치 칸을 클릭합니다. 모든 유닛을 배치한 뒤 Enter를 누르면 전투가 시작됩니다.
5. 붉게 표시된 적을 클릭해 공격하고, Space를 눌러 턴을 종료합니다. 적은 자동으로 이동하고 공격합니다.
6. 배치 완료 직후에는 모든 적이 공격 없이 전투 진입 이동을 한 번 수행한 뒤 첫 플레이어 턴이 시작됩니다.

## 맵 제작 규칙

- `Walkable` Tilemap: 유닛이 설 수 있고 이동할 수 있는 셀
- `Player Deployment` Tilemap: 전투 시작 전 플레이어가 배치할 수 있는 셀
- 이동 목적지는 명령 즉시 예약되므로 여러 유닛이 같은 셀로 이동할 수 없습니다.
- 이동한 칸 수만큼 이동력을 차감하며, 남은 이동력이 있으면 같은 턴에 다시 이동할 수 있습니다.
- 이동 후 공격은 가능하지만 공격을 사용한 뒤에는 남은 이동력과 관계없이 그 턴에 더 이동할 수 없습니다.
- 적은 별도 규칙을 사용해 예고된 공격을 먼저 실행한 뒤 이동합니다. 공격이 빗나가도 예정된 이동은 수행합니다.
- 공격 전에는 이동한 유닛을 다시 선택하고 우클릭해 턴 시작 위치로 복귀하고 이동력을 전부 복구할 수 있습니다. 시작 칸이 점유된 경우에는 취소할 수 없습니다.
- 다른 유닛이 있는 칸은 경로 중간에 통과할 수 있지만 최종 목적지로 선택할 수는 없습니다.
- 유닛 선택 시 이동 범위는 파란색, 현재 위치 기준 공격 범위는 작은 붉은색 칸으로 표시됩니다.
- 공격 사거리는 `AttackRange`(가로)와 `VerticalAttackRange`(세로)를 독립적으로 설정합니다.
- 공격 범위 표시는 이동 범위와 같은 크기의 셀을 사용하며, `VerticalAttackRange`가 2라면 현재 높이에서 위·아래 2칸의 공간 전체가 표시됩니다.
- 플레이어 턴에는 적이 다음 턴에 공격할 타일이 점멸하는 붉은 X로 표시됩니다. 공격은 유닛이 아니라 해당 타일에 고정됩니다.
- 적이 넉백으로 이동하면 공격 방향은 유지되고, 공격 예정 타일도 적이 밀린 거리만큼 함께 이동합니다.
- 적 공격은 팀을 구분하지 않습니다. 공격 예정 타일에 다른 적이 들어가면 해당 적이 피해를 받거나 사망할 수 있습니다.
- 유닛 선택 시 기본적으로 이동 범위만 표시됩니다. 오른쪽 아래 `공격` 버튼을 누르면 공격 범위가 표시되고, 붉은 타일을 좌클릭해 빈 공간이나 아군도 공격할 수 있습니다.
- 씬의 `ATK` 버튼은 공격/공격 취소를 전환하고, `Move Reset` 버튼은 공격 전에 실제로 이동한 유닛을 선택했을 때만 표시되어 이동을 취소합니다.
- 공격 범위에서 대상을 선택해도 즉시 시전되지 않습니다. 주황색은 영향 칸, 노란색은 넉백 도착 칸, 보라색 X는 충돌, 검은색 X는 낙사를 뜻하며 `ATK CONFIRM`으로 확정합니다.
- 공격 모드에서는 `Move Reset` 버튼 위치가 `ATK CANCEL` 버튼으로 바뀌며, 누르면 공격 대상 선택과 공격 모드를 모두 취소합니다.
- 공격 대상을 선택하면 전체 공격 범위는 숨겨지고 타격·넉백·충돌·낙사 미리보기만 남습니다.
- 선택한 유닛 자신의 위치에는 별도의 노란색 하이라이트를 표시하지 않습니다.
- 공격 미리보기에서 실제로 피해를 받을 유닛은 공격 확정 또는 취소 전까지 깜빡이며 표시됩니다.
- `CanPierceUnits`가 꺼진 공격은 중간 유닛 뒤의 칸을 선택할 수 없고, 켜진 공격은 유닛 너머의 칸까지 공격할 수 있습니다.
- 실제 유닛 능력치는 `Assets/StellaStair/Resources/UnitDefinitions`의 `UnitDefinition.asset` 파일들에 저장되며 Inspector에서 직접 수정합니다.
- 마법사는 유닛을 관통해 목표를 선택하며, 명중한 적 주변 1칸의 다른 적들을 중심점 바깥 방향으로 1칸 밀어냅니다.
- 마법사의 주변 넉백은 유닛이 없는 빈 공간을 목표로 지정해도 선택한 칸을 중심으로 정상 발동합니다.
- 마법사의 주변 넉백은 팀을 구분하지 않으며 범위 안의 적, 아군, 시전자 자신을 모두 밀어낼 수 있습니다.
- 공격 미리보기 중에는 직접 피해, 충돌 피해, 낙하 피해와 낙사를 합산해 체력바에서 잃을 구간을 붉게 표시합니다.
- 직접 공격이 치명적이어도 넉백과 벽·유닛 충돌을 먼저 처리한 뒤 사망하므로 마지막 충돌 피해가 정상 적용됩니다.
- 일반 이동도 2칸 이상 아래로 내려가면 각 하강 구간마다 `낙하 칸 수 - 1` 피해를 받으며, 2칸 높이 위쪽 지형은 이동 경로로 사용할 수 없습니다.
- 이동 목적지를 선택하면 즉시 이동하지 않고 예상 낙하 피해를 체력바에 표시하며, `MOVE CONFIRM`으로 확정하거나 `MOVE CANCEL`로 취소합니다.
- `Ladders` Tilemap에 발판 사이를 연속으로 칠하면 같은 열의 위·아래 발판을 연결하며, 사다리 전체 이동은 이동력 1을 소비합니다.
- 사다리를 이용해 내려가는 이동은 낙하로 취급하지 않으므로 낙하 피해가 발생하지 않습니다.
- 사다리 바로 아래의 바닥과 사다리 좌우에 닿은 바닥은 모두 승하차 가능한 이동 지점으로 판정됩니다.
- 사다리 입구나 출구 발판이 점유되면 통과할 수 없으며, 플레이어와 적 AI 모두 같은 사다리 경로 규칙을 사용합니다.
- 사다리 이동은 대각선 이동 대신 진입, 수직 이동, 이탈 순서로 재생됩니다.
- 옆 발판에서 사다리를 탈 때는 실제 사다리 열의 중앙으로 이동한 뒤 수직 이동하므로 발판 벽면을 타고 오르지 않습니다.
- 일반 이동은 1칸 단차까지 오를 수 있으며, 사다리 이동은 사다리 중앙에서 잠시 멈춘 뒤 수직 상승합니다.
- `TacticalBattle` 씬의 오른쪽 버튼은 공격·확인, 왼쪽 버튼은 이동·공격 취소 기능에 직접 바인딩되어 있습니다.
- `KnockbackDistance`가 1 이상인 공격은 대상을 공격 방향으로 밀어냅니다.
- 넉백으로 벽에 충돌하면 대상이 1 피해를 받고, 다른 유닛과 충돌하면 양쪽 모두 1 피해를 받습니다.
- 낮은 바닥으로 떨어지면 `낙하 칸 수 - 1`의 착지 피해를 받으며, 착지할 바닥이 전혀 없는 맵 밖은 낙사 처리됩니다.
- 범위 표시는 지형 타일 내부가 아니라 캐릭터가 서는 바닥 윗면에 표시됩니다.
- 이동과 배치는 표시된 바닥 윗칸을 클릭해야 실행되며 지형 타일 자체의 클릭은 무시됩니다.
- 이웃 열의 가장 높은 표면을 찾아 이동하므로 1칸 단차는 대각선으로 오르고, 같은 열에서 지형을 관통하지 않습니다.
- `UnitDefinition`에서 유닛별 이동력과 실제 이동 속도를 설정합니다.
